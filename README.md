> **"Where half-baked ideas go to rest, and bad code comes to freeze."**
> （優秀點子的安息地，垃圾程式碼的冷凍庫。）

歡迎來到 **VocchiDumbWork**！這裡是我個人的「數位雜物抽屜」兼「程式碼墳墓」。

身為一個工程師，最不缺的就是在深夜衝動開坑、寫了兩行 `console.log` 或一個 `Hello World` 後，隔天就完全失憶的專案。為了保持個人主頁（GitHub Profile）的乾淨與專業，所有**半途而廢、毫無成果、邏輯混亂但又捨不得刪除**的愚蠢 Repo，通通都會被流放到這裡。

---

## 🏗️ 這裡的 Repo 通常具備以下特徵：

*   **完成度極低**：大部分停留在 `init commit`、初期實驗階段，或者卡在某個解不開的 Bug 後被永久棄坑。
*   **技術亂葬崗**：純粹為了踩新雷、試新 Framework 而開的沙盒實驗室。
*   **高含人工智慧量 🤖**：
    > ⚠️ **重要標註**：本組織內有**相當一部分（甚至過半）的專案包含由 AI（如 ChatGPT, Gemini, Claude 等）自動生成的程式碼與設定檔**。
    > 
    > 有些是純 AI 寫的蠢點子，有些是我跟 AI 聊天聊到一半集體崩潰的產物。這裡不保證程式碼的正確性，更不保證它們能夠順利編譯或執行。
*   **嚴禁伸手牌**：這裡的程式碼僅供「歷史考古」與「引以為戒」使用，請勿期待任何 Issue 回覆或 Pull Request 審查。

---

## 🗂️ 墳墓導覽手冊（常見棄坑原因標籤）

如果你在這裡逛街，可能會在各個 Repo 的 Description 看到以下悲傷（但真實）的標籤：

*   `[status: abandoned]` - 寫到一半突然失去熱情。
*   `[reason: re-inventing-the-wheel]` - 寫到第三天發現網路上早就有人做出一模一樣且強大 100 倍的開源套件。
*   `[reason: logic-collapse]` - 邏輯玩太大，把自己繞暈後決定去睡覺。
*   `[ai-assisted: failed-experiment]` - AI 寫得很自信，但我完全跑不起來的產物。

---

## 💡 警語

> **"Enter at your own risk."**
> 這裡沒有聖經，只有反面教材。如果你在這裡找到了你想抄的程式碼，請務必三思，因為連原作者都不知道它為什麼能動（或者為什麼不動）。

感謝你的蒞臨，參觀完請記得輕輕關上墳墓大門。 🚪
